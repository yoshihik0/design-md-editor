# D.mdの使い方

D.mdは、AIとともに`DESIGN.md`を編集する`DESIGN.mdエディター`です。プレビューを見ながら、`DESIGN.md`で記述するデザインシステムをUIで編集するとともに、同じフォルダをAIの作業用フォルダとして指定することで、AIに指示してデザインシステムを提案させたり、編集することができます。

## 2つの起動方法

### サーバーなしで直接開く

`index.html`をChromeまたはEdgeで開きます。初回は「フォルダを選択」を押し、`index.html`が入っているD.mdのプロジェクトフォルダを選んでください。

2回目以降は前回のフォルダを記憶し、権限が残っていれば自動接続します。権限の再確認が必要な場合は「前回のフォルダに再接続」を1回押します。

AIとともに編集する場合は、D.mdのフォルダをCodexやClaude Codeの作業用フォルダとして開きます。

### ローカルサーバーで開く

D.mdのフォルダをCodexやClaude Codeの作業用フォルダとして開きます。

「開いて」と指示すると、ローカルサーバーが立ち上がり、D.mdの画面が表示されます。サーバー版はD.mdのプロジェクトフォルダを自動認識するため、ブラウザでフォルダを選ぶ必要はありません。

サーバーが起動しない場合は、macOSは`start.command`、Windowsは`start.bat`をダブルクリックします。

### 対応ブラウザ（2026年7月24日現在）

- **Chrome／Edge** — 基本的に直接起動版を利用
- **Brave／Firefox／Safari** — ローカルサーバー版を利用
- **Opera／Vivaldiなど** — 直接起動版が動作する可能性はありますが、正式対応外

直接起動版は、ブラウザからローカルフォルダを読み書きするFile System Access APIを使用します。Braveは同じChromium系でもこのAPIを標準では無効にしているため、ローカルサーバー版を利用してください。

## 基本の流れ

1. テンプレートまたは既存の`DESIGN.md`を開く
2. 左側のフォームか右側のYAMLで編集する
3. 中央のプレビューで見た目を確認する
4. 必要なら`PREVIEW.md`の確認項目を編集する
5. AIと相談しながら調整する

ブラウザでの変更は約1秒後に自動保存されます。AIなどがファイルを変更した場合も、約1秒でプレビューへ反映されます。

## AIに依頼するとき

デザインシステムのデータは`design/<デザインシステム名>/`のフォルダのなかの`DESIGN.md`と`PREVIEW.md`の組で管理しています。

AIの処理としては、作業フォルダ内の`.dmd/current.json`を確認して、ユーザーが編集しているデザインシステムを認識します。

現在開いているデザインシステムを変更したい場合は、「もっと鮮やかな色にして」など、変更したい方向性を伝えてください。

新しいデザインシステムを作りたい場合は、「現在のデザインは残したまま、エコ系のデザインシステムを作って」というように指示してください。新しくデザインシステムのフォルダが作られ、`DESIGN.md`と`PREVIEW.md`が生成されます。

新しくデザインシステムを作った場合は、画面は自動では更新されません。「`DESIGN.md`を開く」の選択肢のなかに新しく作られた`DESIGN.md`が選択肢として表示されているはずですので、これを選んでください。

`DESIGN.md`は、色やフォントを定義するだけではなく、その使い方、考え方、デザインに対するフィロソフィーなどを記述することができます。もちろん、根本的なフィロソフィーは人間が決めるべきですが、こういった部分についても、AIにアシストしてもらうことができます。

`DESIGN.md`内の文章もAIが編集する可能性があります。明示的に変更を避けたい範囲は、`<!-- keep -->`〜`<!-- /keep -->`で囲んでください。

## 参考資料を使う

大手企業が公開しているデザインシステムやガイドラインは、記述方法や設計思想の参考になります。既存サイトの`DESIGN.md`や調査メモとともに`references/`へ入れ、AIへ次のように依頼できます。

```text
references/にある既存サイトのDESIGN.mdを参考にしてください。
参考資料自体は変更せず、現在編集中のデザインへ考え方を反映してください。
```

`references/`は参考資料置き場です。

## テンプレートと外部ファイル

テンプレートは選んだだけでは変更されません。最初の編集時に番号付きの新しいデザインシステムのフォルダが作られ、`DESIGN.md`と`PREVIEW.md`が保存されます。

ドラッグ＆ドロップや「その他のファイルを選択」で開いた外部`DESIGN.md`も、元ファイルへ自動保存しません。編集を残す場合は「別名で保存」を使います。

## PREVIEW.md

PREVIEW.mdは、プレビューの表示の仕方をコントロールするファイルです。`DESIGN.md`で使われるカラートークンやタイポグラフィスタイルの名称は、標準パターンはあるものの、かならずしも一定ではありません。さまざまな状況に合わせられるように、プレビューに表示する色の選択や、タイポグラフィーの文例などを変更することができます。

`PREVIEW.md`の記法は、プレビュー上部の`preview.help`で確認できます。

## うまく接続できないとき

- 直接起動ではChromeまたはEdgeを使う
- 前回の権限が切れていたら「前回のフォルダに再接続」を押す
- 違うフォルダを使う場合は上部のフォルダボタンから接続し直す
- フォルダ選択が利用できない場合はサーバー版で起動する
